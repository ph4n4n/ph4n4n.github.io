Những đIều AI Làm đượC Và Ngược Lại (Trong Software Development)

Anph
2025-06-20 (Updated: 2025-06-20)
opinion
ai, opinion, software-development

Mở đầu

AI đang hot như “trend TikTok”, mọi người đều nói về nó. Nhưng thực tế AI trong software development làm được gì và không thể làm được gì? Hôm nay mình sẽ tóm tắt theo kinh nghiệm thực tế (và một chút xạo chó bẳng kiến thức nông cạn của mình) 😄

✅ Những gì AI LÀM ĐƯỢC

1. Code Generation - Viết code cơ bản

AI giỏi:

  • CRUD operations đơn giản
  • Boilerplate code (setup project, config files)
  • Unit tests cơ bản
  • Documentation từ code

Nhưng: AI viết code như “sinh viên năm 2” - syntax đúng nhưng thiếu tối ưu và edge cases 🎓

2. Code Review - Tìm lỗi cơ bản

AI phát hiện được:

  • Syntax errors
  • Code smells đơn giản
  • Security vulnerabilities phổ biến
  • Performance issues cơ bản

Ví dụ AI bắt được:

3. Debugging - Giải thích lỗi

AI khá giỏi đọc stack trace và đưa ra gợi ý. Giống như “Stack Overflow thông minh hơn” vậy.

4. Code Refactoring - Cải thiện code

AI có thể:

  • Extract functions
  • Rename variables có nghĩa
  • Apply design patterns cơ bản
  • Convert giữa các coding styles

5. Learning & Research - Tra cứu nhanh

AI = Google + Stack Overflow + Documentation hợp thể:

  • Giải thích concepts
  • So sánh technologies
  • Đưa ra examples
  • Tóm tắt best practices

❌ Những gì AI KHÔNG LÀM ĐƯỢC

1. Hiểu Business Logic phức tạp

AI không hiểu:

  • Tại sao discount 15% cho customer VIP nhưng 10% cho regular?
  • Logic nghiệp vụ đặc thù của từng công ty
  • Requirements ẩn không được viết ra

2. Architecture Design - Thiết kế hệ thống

AI không thể:

  • Chọn tech stack phù hợp với team và budget
  • Design scalable architecture
  • Predict future requirements
  • Balance trade-offs (performance vs maintainability)

Ví dụ: AI không biết khi nào nên dùng microservices vs monolith. Nó chỉ biết copy-paste pattern mà không hiểu context.

3. Code Review sâu - Hiểu impact

AI bỏ lỡ:

  • Performance impact trong production
  • Maintainability issues
  • Team coding standards đặc thù
  • Domain knowledge requirements

4. Debugging phức tạp - Root cause analysis

AI không thể:

  • Debug race conditions
  • Tìm memory leaks
  • Phân tích performance bottlenecks
  • Hiểu distributed system issues

5. Creative Problem Solving

AI thiếu:

  • Out-of-the-box thinking
  • Innovation và creativity
  • Adaptation to unique constraints
  • Intuition từ kinh nghiệm

6. Team Collaboration & Communication

AI không thể:

  • Negotiate requirements với stakeholders
  • Mentor junior developers
  • Handle conflicts trong team
  • Present technical solutions to non-tech people

7. Production Operations

AI yếu trong:

  • Incident response và troubleshooting
  • Capacity planning
  • Security assessment
  • Compliance requirements

🤔 Vùng “MAYBE” - Phụ thuộc context

1. Testing Strategy

  • Làm được: Unit tests cơ bản
  • Không được: Integration testing strategy, test scenarios phức tạp

2. API Design

  • Làm được: REST endpoints cơ bản
  • Không được: API versioning strategy, backward compatibility

3. Database Design

  • Làm được: Simple schema
  • Không được: Performance optimization, complex relationships

💡 Cách sử dụng AI hiệu quả

✅ DO:

  • Dùng AI như “smart assistant”
  • Generate boilerplate code
  • Explain unfamiliar concepts
  • Quick prototyping
  • Code review bước đầu

❌ DON’T:

  • Tin 100% vào AI code
  • Để AI quyết định architecture
  • Copy-paste mà không hiểu
  • Thay thế hoàn toàn human review
  • Dùng AI cho production-critical code

Kết luận

AI giống như “Junior Developer siêu thông minh”:

  • Viết code nhanh, syntax đúng
  • Nhớ được nhiều thứ
  • Làm việc 24/7 không than phiền
  • NHƯNG thiếu kinh nghiệm, context và wisdom

Công thức thành công:

Lời khuyên cuối:

  • AI là tool, không phải replacement
  • Học cách collaborate với AI
  • Đừng để AI làm mình “lười suy nghĩ”
  • Keep learning, AI cũng đang học 📚

Remember: AI có thể code được, nhưng không thể “cảm” được deadline như bạn! 😅
Remember2: AI có thể code được, nhưng không thể buồn khi bị sếp chửi, đồng nghiệp quay lưng như bạn! 😅
Remember3: AI có thể code được, nhưng không thể lười như bạn! 😅

AI thời đại mới, dev vẫn quan trọng nhất!
